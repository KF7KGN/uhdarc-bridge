What UHDARC built: A radio convergence hub that acts like a central telephone switchboard for digital radio networks. Four completely incompatible digital radio protocols — DMR, P25, YSF, and EchoLink — are all bridged through a single Asterisk/AllStar node. A person on a DMR radio and a person on a P25 radio on opposite sides of the country can talk to each other in real time through this system.

Why this is hard: DMR, P25, YSF, and EchoLink all use completely different audio compression, signaling, and network protocols. They cannot natively communicate with each other. This system transcodes audio between all four modes simultaneously using AMBE encoding, MMDVM bridging, and Asterisk IAX2 routing — all running in production on a single Debian Linux server.

The Four Digital Modes

DMR — Digital Mobile Radio (Node 1999)

The most widely used digital radio protocol worldwide. Uses TDMA (Time Division Multiple Access) to put two conversations on one frequency simultaneously using two time slots. Connected via HBlink3 master server to DMR networks across the USA.

Central Hub — Asterisk/AllStar Node 573470

What it is: A specialized version of Asterisk modified for amateur radio called AllStar. It acts as the central switching matrix routing audio between all four digital modes via IAX2 protocol. When someone transmits on any mode, Asterisk receives the audio, determines routing, and sends it to all connected networks simultaneously.

Complete Audio Flow — End to End

STEP 1 — TRANSMISSION DMR Radio transmits → USRP Receiver listens on port 34001 → Audio enters Asterisk Node 1999 STEP 2 — ROUTING Asterisk Node 1999 → IAX2 connects to Node 573470 (hub) → Hub routes to all connected bridges STEP 3 — TRANSCODING Analog_Bridge receives PCM audio → AMBE Encoder compresses audio (port 31103) → Compressed AMBE frames sent to HBlink3 STEP 4 — NETWORK DISTRIBUTION HBlink3 Master (port 62030) ├── DMR Peer 1 (remote — IP redacted) ├── DMR Peer 2 (remote — IP redacted) ├── DMR Peer 3 (remote — IP redacted) └── 20+ more peers nationwide MMDVM_Bridge simultaneously ├── P25Gateway → Quantar (port 1994) └── YSFGateway → YSF Reflectors (port 42002) theBridge simultaneously └── EchoLink Conference (port 8443) STEP 5 — RETURN AUDIO Remote transmission arrives at HBlink3 → HBlink3 → Asterisk (IAX2) → Asterisk → Analog_Bridge → AMBE Decoder (port 31100) → PCM Audio → USRP TX (port 32001) → Local speaker / transmitter RESULT Everyone on DMR, P25, YSF, and EchoLink all hear the same transmission simultaneously regardless of what radio or protocol they use

Architecture Notes

Single Point of Failure: All 9 services, all 4 digital modes, and all 75 channels run on one server. If the server goes down the entire UHDARC bridge stops. Recommended improvement: geographic redundancy with a backup server and automatic failover.
What is working well:
  • Asterisk stable — 2+ day runs without restart
  • All 4 digital modes bridging successfully in production
  • 20+ remote IAX2 peers staying connected
  • Audio quality tuned with proper gain and AGC settings per node
  • Full monitoring stack — Allmon3, HBmonitor, custom API dashboard
  • Security — TLS on Apache, reCAPTCHA, Tailscale VPN, UFW firewall
  • Automated backups, daily service reloads, systemd timers
